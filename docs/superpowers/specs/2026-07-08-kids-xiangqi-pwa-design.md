# 儿童象棋 PWA 设计说明

## 背景与目标

为 iPad 创建一款儿童友好的中国象棋游戏。小朋友通过 Safari 打开 HTTPS 静态站点并添加到主屏幕后，可以离线直接玩。第一版只做本地人机对弈，不做账号、联网对战、排行榜、残局关卡或后端服务。

## 目标用户

- 家里正在学习或练习象棋的小朋友。
- 家长希望游戏不依赖局域网服务，安装和使用都简单。
- 同一套界面既能辅助入门，也能在关闭提示后当标准象棋练习使用。

## 功能范围

- 独立静态 PWA，可部署到 GitHub Pages、Vercel 或 Netlify 这类 HTTPS 静态站点。
- iPad Safari 支持添加到主屏幕，资源缓存后可离线打开。
- 本地人机对弈，玩家默认执红先行，AI 执黑。
- 三档难度：
  - 入门：AI 使用浅层评估并有轻微随机性，教学提示默认开启，适合刚认识规则的小朋友。
  - 初级：AI 使用一层搜索和基础吃子/防守评估，提示仍可用但不强打扰。
  - 进阶：AI 使用更深一点的 minimax/alpha-beta 搜索和局面评估，形成基础挑战。
- 规则校验：
  - 标准 9x10 象棋棋盘。
  - 支持帅、仕、相、马、车、炮、兵的合法走法。
  - 禁止走完后己方被将军。
  - 支持将帅照面规则。
  - 识别将军、被将死、无合法走法失败。
- 儿童友好交互：
  - 点击棋子后高亮可走位置。
  - 非法点击给出温和提示。
  - 可悔棋：一次悔棋撤销玩家一步和 AI 一步。
  - 可提示一步：根据当前难度给出推荐走法。
  - 教学提示可关闭。
  - 新开一局、切换难度、切换提示开关。
- 本地持久化：
  - 保存难度、教学提示开关和最近对局。
  - PWA 离线时继续可用。

## 非目标

- 不做在线匹配、局域网服务、WebSocket、账号体系、云端存档。
- 不做完整棋谱库、开局库、残局闯关。
- 不接入外部象棋引擎或服务端 AI。
- 不追求专业象棋引擎强度，优先保证儿童可玩性、稳定性和离线体验。

## 技术方案

采用 React 18 + TypeScript + Vite 5 创建静态前端项目，使用 `vite-plugin-pwa` 生成 manifest 和 service worker。象棋规则引擎、AI 和 UI 状态全部在浏览器本地运行。项目使用 Vitest 覆盖规则、AI 和存档逻辑。版本选择以当前本机 Node 18.17.1 可稳定安装和构建为准。

不引入 GPL 象棋规则库。npm 上可用的 MIT 象棋包要么 API 过老，要么文档和维护状态不足以支撑儿童产品的规则稳定性；第一版选择在项目内实现规则引擎，并用测试覆盖核心规则。

## 模块边界

- `src/game/types.ts`：棋子、坐标、走法、游戏状态、难度等类型。
- `src/game/setup.ts`：初始棋盘和局面创建。
- `src/game/rules.ts`：基础走法生成、合法走法过滤、将军/照面/胜负判定。
- `src/game/engine.ts`：应用走法、撤销栈、提示文案需要的局面摘要。
- `src/game/ai.ts`：三档难度配置、局面评估、AI 选步。
- `src/storage/preferences.ts`：localStorage 存取难度、教学开关、最近局面。
- `src/App.tsx`：主流程编排，连接棋盘、控制栏、状态栏和提示面板。
- `src/components/Board.tsx`：棋盘渲染、棋子点击、高亮和移动目标。
- `src/components/Controls.tsx`：难度、提示开关、新局、悔棋、提示按钮。
- `src/components/StatusPanel.tsx`：当前回合、将军、胜负和儿童提示。
- `public/manifest.webmanifest` 与 PWA 图标：主屏幕安装和离线缓存。

## 数据流

1. App 初始化时从 localStorage 读取偏好和最近局面；读取失败时使用默认新局。
2. 玩家点击棋子，Board 请求 rules 生成该棋子的合法目标并高亮。
3. 玩家点击目标，engine 应用走法并把快照写入历史。
4. 若游戏未结束，App 触发 AI 计算一步，engine 应用 AI 走法。
5. App 将新局面、难度和提示开关保存到 localStorage。
6. 悔棋时撤销最后两步，保证回到玩家回合。

## UI 与体验

- iPad 横屏优先，纵屏可用。
- 第一屏就是棋盘和控制区，不做营销落地页。
- 棋盘使用木色棋盘、红黑棋子、清晰中文棋名，按钮使用图标加短中文标签。
- 棋盘尺寸自适应，保持 9x10 比例，不能因为提示文字或按钮状态造成布局跳动。
- 状态提示使用短句，例如“红方走棋”“黑方将军”“这一步会让帅被将军”。
- 教学提示关闭后只保留必要状态，不显示冗长解释。

## AI 设计

- 入门：从合法走法中基于轻量评分和随机扰动选步，倾向于不总是吃最贵棋子，降低挫败感。
- 初级：一层搜索，优先安全吃子、防守被攻击高价值棋子、避免立即送将。
- 进阶：两层或三层 alpha-beta 搜索，使用棋子价值、过河兵奖励、中心控制、将军奖励和己方被威胁惩罚。
- 所有难度都必须只从合法走法中选择，不允许让 AI 走出非法棋。

## 错误处理与边界

- localStorage 解析失败时丢弃坏数据并新开一局。
- PWA service worker 更新失败不阻断游戏。
- 当前局面没有合法走法时立即进入结束状态。
- 提示一步时如果无合法走法，显示结束状态而不是抛错。
- AI 计算期间禁用玩家落子，避免重复点击造成状态错乱。

## 测试与验证

- 单元测试覆盖：
  - 初始棋盘棋子数量和关键位置。
  - 车、马、炮、兵、帅/将、仕、相的基础合法走法。
  - 蹩马腿、塞象眼、炮隔山打子、兵过河、九宫限制。
  - 将帅照面和不能走完后被将军。
  - 将军/将死识别。
  - AI 三档均返回合法走法。
  - 悔棋撤销玩家和 AI 两步。
  - 偏好存储坏数据降级。
- 构建验证：
  - `npm test`
  - `npm run build`
  - 本地预览 PWA，使用浏览器检查 iPad 尺寸下棋盘非空、可点击、可离线缓存。

## 部署说明

第一版只交付静态产物。构建命令为 `npm run build`，输出目录为 `dist/`。部署到 GitHub Pages、Vercel 或 Netlify 时只需要配置静态目录，不需要后端环境变量。HTTPS 托管完成后，在 iPad Safari 打开站点并添加到主屏幕即可。

## 发布与回归范围

- 回归三档难度能正常开始并完成至少数步。
- 关闭网络后从主屏幕重新打开仍可进入游戏。
- 棋盘点击、高亮、悔棋、提示、新局、切换难度均可用。
- iPad 横屏和竖屏无文字重叠，按钮文字不溢出。
- 切换版本后 service worker 能更新缓存，必要时提示刷新。
